# Day 1: Bibliography & References

Here is a curated collection of seminal whitepapers, official documentation, engineering blogs, and learning guides to deepen your understanding of modern data platform architectures.

---

## 📄 Seminal Big Data Research Papers

1. **The Google File System (GFS) (2003)**
   - *Authors:* Sanjay Ghemawat, Howard Gobioff, and Shun-Tak Leung.
   - *Significance:* Outlined the architecture of a scale-out distributed file system which directly inspired Apache Hadoop HDFS.
   - *Link:* [GFS Research Paper PDF](https://static.googleusercontent.com/media/research.google.com/en//archive/gfs-sosp2003.pdf)

2. **MapReduce: Simplified Data Processing on Large Clusters (2004)**
   - *Authors:* Jeffrey Dean and Sanjay Ghemawat.
   - *Significance:* Introduced the functional programming paradigm (Map and Reduce) for parallel processing across commodity clusters.
   - *Link:* [MapReduce Research Paper PDF](https://static.googleusercontent.com/media/research.google.com/en//archive/mapreduce-osdi04.pdf)

3. **Bigtable: A Distributed Storage System for Structured Data (2006)**
   - *Authors:* Fay Chang, Jeffrey Dean, Sanjay Ghemawat, et al.
   - *Significance:* Outlined wide-column Key-Value storage, laying the blueprint for Apache HBase and Apache Cassandra.
   - *Link:* [Bigtable Research Paper PDF](https://static.googleusercontent.com/media/research.google.com/en//archive/bigtable-osdi06.pdf)

## 📖 Recommended Books

* **"Designing Data-Intensive Applications" (DDIA)** by Martin Kleppmann. (The absolute gold standard for understanding distributed systems, replication, partitioning, and consensus).
* **"Hadoop: The Definitive Guide"** by Tom White. (Essential for deep-diving into Hadoop internals, HDFS block structures, and MapReduce flows).
* **"Spark: The Definitive Guide"** by Bill Chambers and Matei Zaharia. (Best guide for Spark SQL, structured streaming, and API mechanics).
