# Hands-On Lab: Run a Dockerized Workload on YARN

This hands-on lab guides you through configuring, building, and running a Dockerized application on an Apache Hadoop YARN cluster.

---

## 🔬 Lab Prerequisites

To execute this lab, ensure you have:
1. A Linux-based environment (or Docker Desktop on Windows/macOS running WSL2).
2. Docker Engine (version 19.03+) and Docker Compose installed.
3. Access to terminal commands with `sudo` permissions (required for configuring local `container-executor`).

---

## 🚀 Step 1: Clone and Start the Local Cluster

First, navigate to the docker directory for Day 8 and bootstrap the containerized multi-node cluster:

```bash
# Navigate to directory
cd docker/

# Start HDFS and YARN containers
docker-compose up -d

# Verify that all containers are running successfully
docker-compose ps
```

*Expected Output:*
You should see four containers running: `namenode-day08`, `datanode-day08`, `resourcemanager-day08`, and `nodemanager-day08`.

---

## ⚙️ Step 2: Configure YARN NodeManager for Docker Runtime

We need to make sure the YARN NodeManager has access to the configuration files. In this docker-compose setup, these are pre-mounted. Let's log in to the NodeManager to verify.

```bash
# Log into the NodeManager container as root
docker exec -it nodemanager-day08 bash

# Validate the yarn-site.xml contains docker properties
grep -A 2 "yarn.nodemanager.runtime.linux.allowed-runtimes" /etc/hadoop/yarn-site.xml
```

*Expected Output:*
```xml
<name>yarn.nodemanager.runtime.linux.allowed-runtimes</name>
<value>default,docker</value>
```

Verify that the local security configuration is correct inside `/etc/hadoop/container-executor.cfg`:
```bash
cat /etc/hadoop/container-executor.cfg
```

---

## 🏗️ Step 3: Build the Sample Docker Application

We will build the sample Python-based workload container that will be launched by YARN. On your host machine:

```bash
# Navigate back to the root of Day-08 directory
cd ..

# Build the sample application docker image
docker build -t yarn-sample-app:latest examples/docker-app/
```

Verify that the image was built successfully and is in the local daemon cache:
```bash
docker images | grep yarn-sample-app
```

---

## 🚀 Step 4: Submit the Containerized Application to YARN

We submit our containerized application using YARN DistributedShell. Since NodeManager shares the host's Docker socket, the YARN NodeManager will execute a `docker run` command pulling from the host's local cache.

Execute the following command from the host:
```bash
# Run the demo submission script
bash scripts/run-docker-demo.sh
```

If you are running manually inside the `resourcemanager-day08` container, run:
```bash
# Execute within the ResourceManager container
docker exec -it resourcemanager-day08 yarn jar \
  /opt/hadoop/share/hadoop/yarn/hadoop-yarn-applications-distributedshell-2.0.0-hadoop3.2.1-java8.jar \
  -appname docker-yarn-demo \
  -shell_command "python3 /app/process.py" \
  -num_containers 1 \
  -container_memory 1024 \
  -container_vcores 1 \
  -shell_env YARN_CONTAINER_RUNTIME_TYPE=docker \
  -shell_env YARN_CONTAINER_RUNTIME_DOCKER_IMAGE=yarn-sample-app:latest \
  -shell_env YARN_CONTAINER_RUNTIME_DOCKER_RUN_OVERRIDE_DISABLE=true
```

---

## 📊 Step 5: Verify Execution and Monitor UI

1. Open your browser and navigate to the YARN ResourceManager Web UI at **[http://localhost:8088](http://localhost:8088)**.
2. Search for the application named `docker-yarn-demo`.
3. Check the state of the application. Once it is marked as `FINISHED` with final status `SUCCEEDED`, click on the application ID.

## 🧹 Step 7: Cleanup

Once you complete the lab, clean up the cluster resources:

