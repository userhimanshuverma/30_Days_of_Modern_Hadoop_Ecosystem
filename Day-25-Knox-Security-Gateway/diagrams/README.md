# Day 25 Diagrams — Apache Knox Gateway Architecture

This folder contains architectural and execution flow diagrams for the Apache Knox Security Gateway. The diagrams are written in Mermaid markdown syntax and rendered directly in the course documentation.

## 🗺️ List of Diagrams

### 1. Perimeter Security Architecture
Shows how Apache Knox acts as a single gatekeeper (reverse proxy) in front of the Hadoop cluster services (HDFS, Hive, YARN, Spark), shielding them from direct client access.
- **File Reference:** Included in main [README.md](../README.md#31-architecture-overview)

### 2. Request and Dispatch Flow (URL Rewriting & Proxy)
Illustrates how an HTTP request travels from the client, undergoes authentication, authorization, and URL rewriting, before being dispatched to the actual backend Hadoop node.
- **File Reference:** Included in main [README.md](../README.md#41-request-execution-flow)

### 3. LDAP Authentication Sequence Flow
Focuses on the interaction between the Knox Gateway and the identity provider (OpenLDAP / Active Directory) to authenticate the client credentials before allowing access to the resource.
- **File Reference:** Included in main [README.md](../README.md#32-ldap-authentication-flow)

### 4. HDFS 307 Redirect Flow (WebHDFS write rewriting)
Highlights how Knox intercepts HTTP redirects (307 Temporary Redirects) generated by HDFS NameNode pointing to HDFS DataNode, rewrites them to route through Knox, and passes the payload.
- **File Reference:** Included in main [README.md](../README.md#56-url-rewriting-mechanics)
